## Overview

The interactive calendar provides a visual, clickable interface to manage security guard schedules. Each day displays its shift type, applicable divisor, and calculated pay. Click any day to open a detailed modal with breakdown calculations and editing options.

## Calendar Layout

### Grid Structure

```javascript index.html (lines 456-493)
function renderGrid(year, month) {
    const grid = document.getElementById('calendarGrid');
    grid.innerHTML = '';
    const firstDayOfMonth = new Date(year, month - 1, 1);
    const startDayOfWeek = firstDayOfMonth.getDay(); 
    const daysInMonth = dayStates.length - 1;

    // Empty cells for alignment
    for (let i = 0; i < startDayOfWeek; i++) {
        const empty = document.createElement('div');
        empty.className = 'day-cell empty';
        grid.appendChild(empty);
    }

    // Render each day
    for (let day = 1; day <= daysInMonth; day++) {
        const state = dayStates[day];
        const cell = document.createElement('button');
        cell.setAttribute('type', 'button');
        const config = stateConfig[state];
        const divisor = getDivisorForDate(year, month, day);
        const cellVals = calculateDayComponents(state, year, month, day); 

        cell.id = `day-${day}`;
        cell.className = `day-cell ${config.class}`;
        
        const moneyText = Math.round(cellVals.total).toLocaleString();
        cell.setAttribute('aria-label', `Día ${day}, ${config.label}, Valor: $${moneyText}. Haz clic para detalles.`);
        
        cell.onclick = () => openModal(day);

        cell.innerHTML = `
            <div class="day-number">${day} <span class="rate-indicator">${divisor}h</span></div>
            <div class="day-label">${config.label}</div>
            <div class="day-money">$ ${Math.round(cellVals.total).toLocaleString()}</div>
        `;
        grid.appendChild(cell);
    }
}
```

**Features**:
- **7-day week layout** with Sunday-Saturday header
- **Empty cells** for proper calendar alignment
- **Color-coded** shift types via CSS classes
- **Accessible** with ARIA labels and button semantics

## Interactive Modal System

### Opening the Modal

```javascript index.html (lines 496-522)
function openModal(day) {
    currentSelectedDay = day;
    const year = parseInt(document.getElementById('selYear').value);
    const month = parseInt(document.getElementById('selMonth').value);
    
    // 1. Title
    const months = ["Enero","Febrero","Marzo","Abril","Mayo","Junio",
                    "Julio","Agosto","Septiembre","Octubre","Noviembre","Diciembre"];
    document.getElementById('modalDateTitle').innerText = `Día ${day} de ${months[month-1]} ${year}`;
    
    // 2. Law Info
    const divisor = getDivisorForDate(year, month, day);
    const salary = getRawMoney('baseSalary');
    const hourValue = Math.round(salary / divisor);
    document.getElementById('modalLawInfo').innerText = 
        `Divisor ${divisor} horas | Valor Hora: ${formatMoney(hourValue)}`;

    // 3. Pre-select current state
    const currentState = dayStates[day];
    document.getElementById('modalShiftSelector').value = currentState;

    // 4. Calculate and display
    updateModalCalculations(year, month, day, currentState);

    // 5. Show modal
    document.getElementById('dayModal').style.display = 'flex';
    document.getElementById('modalShiftSelector').focus();
}
```

**Modal displays**:
1. **Date title**: "Día 15 de Marzo 2026"
2. **Legal context**: Divisor and hour value for that specific date
3. **Shift selector**: Dropdown to change shift type
4. **Breakdown table**: Basic/Recargos/Extras/Total

### Modal Breakdown Table

```javascript index.html (lines 549-561)
function updateModalCalculations(year, month, day, state) {
    const comps = calculateDayComponents(state, year, month, day);
    
    document.getElementById('tdBasicVal').innerText = formatMoney(comps.basic);
    document.getElementById('tdRecVal').innerText = formatMoney(comps.recargo);
    document.getElementById('tdExtVal').innerText = formatMoney(comps.extra);
    document.getElementById('tdTotalVal').innerText = formatMoney(comps.total);

    const desc = getDescriptions(state);
    document.getElementById('tdBasicDesc').innerText = desc.basic;
    document.getElementById('tdRecDesc').innerText = desc.recargo;
    document.getElementById('tdExtDesc').innerText = desc.extra;
}
```

**Table columns**:
1. **Concepto**: Type of pay (Básico, Recargos, Extras)
2. **Cálculo**: Description of calculation (e.g., "8h Ordinarias")
3. **Valor**: Monetary amount

### Description Logic

```javascript index.html (lines 563-573)
function getDescriptions(state) {
    switch(state) {
        case 'D': return { basic: '8h Ordinarias', recargo: '0', extra: '4h Extras Diurnas (1.25)' };
        case 'N': return { basic: '1h Diu + 7h Noc', recargo: '7h Recargo Noc (0.35)', extra: '4h Extra Noc (1.75)' };
        case 'X': return { basic: '8h Pago Básico', recargo: '0', extra: '0' };
        case 'E': return { basic: '8h Base', recargo: '8h Recargo Festivo', extra: '4h Extra Festiva' };
        case 'P': return { basic: '8h Permiso Pago', recargo: '0', extra: '0' };
        case 'A': return { basic: '0', recargo: '0', extra: '0' };
        default: return { basic: '-', recargo: '-', extra: '-' };
    }
}
```

**Example for Night Shift**:
- **Básico**: "1h Diu + 7h Noc" = $63,672
- **Recargos**: "7h Recargo Noc (0.35)" = $19,498
- **Extras**: "4h Extra Noc (1.75)" = $55,714
- **Total**: $138,884

## Real-Time Recalculation

### Update Chain

```javascript index.html (lines 418-424)
function fullRecalculate() {
    updateEducationalTable();
    recalculate();
    const year = parseInt(document.getElementById('selYear').value);
    const month = parseInt(document.getElementById('selMonth').value);
    renderGrid(year, month);
}
```

**Triggered by**:
- Changing salary or transport fields
- Switching shift pattern or cycle start date
- Toggling commercial/real month mode
- Editing individual days in modal

**Updates**:
1. **Educational table**: Hour value and rate cards
2. **Monthly totals**: Loop through all days, sum components
3. **Calendar grid**: Re-render all cells with new values

<Note>
  All calculations happen **client-side** in JavaScript - no server requests. Updates are instant.
</Note>

## Performance Optimization

### Incremental Rendering

The grid renders only visible days (28-31), not the entire year. Pattern generation loops only through current month.

### Calculation Caching

`calculateDayComponents()` is called per-day but uses constant-time lookups (no nested loops). Monthly totals loop once through 28-31 days.

### Minimal DOM Updates

On single-day edit:
1. Only that day's cell re-renders
2. Summary section updates (4-5 DOM writes)
3. Modal table updates (4 rows)

Total: ~10 DOM operations vs full re-render.
